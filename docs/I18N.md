# Інтернаціоналізація (i18n) - Інструкція для розробників

## Огляд

Проєкт використовує **Lingui** для інтернаціоналізації. Переклади зберігаються у TypeScript файлах у папці `src/locales/`.

### Підтримувані мови

- `uk` - Українська (за замовчуванням)
- `en` - English

---

## Структура файлів локалізації

```
src/locales/
├── en/
│   ├── create.ts      # Переклади для сторінок (request.create.*, request.list.* тощо)
│   └── metadata.ts    # Метадані сторінок (SEO)
├── uk/
│   ├── create.ts      # Переклади для сторінок
│   └── metadata.ts    # Метадані сторінок
├── i18n.ts            # Ініціалізація Lingui
├── locale.ts          # Утиліти для визначення локалі
└── route-metadata.ts  # Метадані маршрутів
```

## Формат файлів перекладу

Кожен файл перекладу експортує об'єкт `messages` типу `Messages`:

```typescript
// src/locales/uk/create.ts
import type { Messages } from '@lingui/core';

export const messages: Messages = {
  // Коментар для групування
  'namespace.section.key': 'Переклад українською',
  'namespace.section.anotherKey': 'Інший переклад',
};
```

### Конвенція іменування ключів

Використовуйте формат: `namespace.section.key`

| Частина | Опис | Приклад |
|---------|------|---------|
| `namespace` | Загальна область (entity або feature) | `request`, `user`, `auth` |
| `section` | Конкретна сторінка або компонент | `create`, `list`, `detail` |
| `key` | Конкретний елемент UI | `title`, `subtitle`, `submitButton` |

**Приклади:**

```typescript
// Сторінка створення запиту
'request.create.title': 'Створити запит',
'request.create.submitButton': 'Опублікувати',

// Сторінка списку запитів
'request.list.title': 'Всі запити',
'request.list.searchPlaceholder': 'Пошук запитів...',

// Сторінка деталей запиту
'request.detail.contactButton': 'Зв\'язатися',
```

## Як додати переклад до нового компонента

### Крок 1: Імпортувати `useLingui`

```typescript
'use client';

import { useLingui } from '@lingui/react';
```

### Крок 2: Створити функцію `t` в компоненті

```typescript
export default function MyComponent() {
  const { i18n } = useLingui();
  const t = (id: string) => i18n._(id);

  return (
    <div>
      <h1>{t('namespace.section.title')}</h1>
      <p>{t('namespace.section.description')}</p>
    </div>
  );
}
```

### Крок 3: Додати ключі перекладу у файли локалей

**Українська (`src/locales/uk/create.ts`):**

```typescript
export const messages: Messages = {
  // ... existing translations
  
  // My new component
  'namespace.section.title': 'Заголовок українською',
  'namespace.section.description': 'Опис українською',
};
```

**Англійська (`src/locales/en/create.ts`):**

```typescript
export const messages: Messages = {
  // ... existing translations
  
  // My new component
  'namespace.section.title': 'Title in English',
  'namespace.section.description': 'Description in English',
};
```

## Повний приклад

### До (без перекладу):

```typescript
'use client';

export default function RequestList() {
  return (
    <div>
      <h1>Знайдіть своїх клієнтів</h1>
      <p>Переглядайте запити та пропонуйте свої послуги</p>
      <input placeholder="Пошук запитів..." />
      <button>Завантажити ще</button>
    </div>
  );
}
```

### Після (з перекладом):

```typescript
'use client';

import { useLingui } from '@lingui/react';

export default function RequestList() {
  const { i18n } = useLingui();
  const t = (id: string) => i18n._(id);

  return (
    <div>
      <h1>
        {t('request.list.titlePrefix')}{' '}
        <span>{t('request.list.titleEmphasis')}</span>
      </h1>
      <p>{t('request.list.subtitle')}</p>
      <input placeholder={t('request.list.searchPlaceholder')} />
      <button>{t('request.list.loadMore')}</button>
    </div>
  );
}
```

## Чеклист для додавання перекладів

- [ ] Імпортувати `useLingui` з `@lingui/react`
- [ ] Створити функцію `t` через `i18n._`
- [ ] Замінити всі захардкоджені тексти на `t('key')`
- [ ] Додати ключі в `src/locales/uk/create.ts`
- [ ] Додати ключі в `src/locales/en/create.ts`
- [ ] Перевірити, що ключі унікальні та відповідають конвенції іменування
- [ ] Групувати пов'язані ключі за допомогою коментарів

## Типові помилки

### 1. Забули додати `'use client'`

```typescript
// Помилка: useLingui працює тільки в клієнтських компонентах
import { useLingui } from '@lingui/react';

export default function MyComponent() {
  const { i18n } = useLingui(); // Error!
}
```

**Рішення:** Додати `'use client';` на початку файлу.

### 2. Неконсистентні ключі

```typescript
// Погано: різні формати ключів
'request.create.title': '...',
'requestList.title': '...',      // Неконсистентно!
'request-detail-title': '...',   // Неконсистентно!
```

**Рішення:** Завжди використовувати формат `namespace.section.key` з крапками.

### 3. Дублювання ключів

```typescript
// Помилка: один ключ для різних значень
// В uk/create.ts
'button.submit': 'Надіслати',

// В uk/create.ts (інше місце)
'button.submit': 'Зберегти',  // Перезапише попередній!
```

**Рішення:** Використовувати унікальні ключі: `form.submitButton`, `settings.saveButton`.

### 4. Забули додати переклад для обох мов

```typescript
// uk/create.ts
'request.new.title': 'Новий запит',

// en/create.ts
// Забули додати! Буде показано ключ замість тексту
```

**Рішення:** Завжди додавати переклади в обидва файли одночасно.

## Де використовувати переклади

| Елемент | Чи потрібен переклад? |
|---------|----------------------|
| Заголовки (h1, h2, тощо) | Так |
| Тексти параграфів | Так |
| Placeholder в input | Так |
| Тексти кнопок | Так |
| Підказки (tooltip) | Так |
| Повідомлення про помилки | Так |
| Альтернативний текст (alt) | Так |
| aria-label | Так |
| Назви класів CSS | Ні |
| ID елементів | Ні |
| Ключі об'єктів | Ні |
| URL адреси | Ні |

## Спільні переклади (переіспользувані)

Якщо переклад використовується в кількох місцях, використовуйте спільний namespace:

```typescript
// Спільні переклади
'common.loading': 'Завантаження...',
'common.error': 'Сталася помилка',
'common.save': 'Зберегти',
'common.cancel': 'Скасувати',
'common.delete': 'Видалити',
'common.edit': 'Редагувати',
'common.back': 'Назад',
'common.next': 'Далі',

// Форми
'form.required': "Це поле є обов'язковим",
'form.invalidEmail': 'Невірний формат email',
```

## Динамічні значення в перекладах

Для вставки динамічних значень використовуйте шаблонні рядки:

```typescript
// Переклад з placeholder
'request.list.found': 'Знайдено {count} запитів',

// Використання
const count = 10;
const text = t('request.list.found').replace('{count}', String(count));
// Результат: "Знайдено 10 запитів"
```

Або можна розділити на частини:

```typescript
// Переклади
'request.list.foundPrefix': 'Знайдено',
'request.list.foundSuffix': 'активних запитів',

// Використання в JSX
<>
  {t('request.list.foundPrefix')}{' '}
  <span>{count}</span>{' '}
  {t('request.list.foundSuffix')}
</>
```

## Підсумок

1. **Імпортувати** `useLingui` та створити функцію `t`
2. **Замінити** всі тексти на виклики `t('key')`
3. **Додати** ключі в обидва файли локалізації (`uk` та `en`)
4. **Дотримуватися** конвенції іменування: `namespace.section.key`
5. **Групувати** пов'язані ключі коментарями
