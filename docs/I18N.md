# Інтернаціоналізація (i18n) - Інструкція для розробників

## Огляд

Проєкт використовує **LinguiJS** для інтернаціоналізації та **Next.js** для рендерингу (згідно з архітектурою FSD, див. `@docs/ARCHITECTURE.md`). Переклади зберігаються у TypeScript файлах у папці `src/locales/`.

### Підтримувані мови

- `uk` - Українська (за замовчуванням)
- `en` - English

---

## Структура файлів локалізації

```
src/locales/
├── en/
│   ├── create.ts      # Переклади для сторінок (request.create.*, request.list.* тощо)
│   └── metadata.ts    # Метадані сторінок (SEO)
├── uk/
│   ├── create.ts      # Переклади для сторінок
│   └── metadata.ts    # Метадані сторінок
├── i18n.ts            # Ініціалізація Lingui
├── locale.ts          # Утиліти для визначення локалі
└── route-metadata.ts  # Метадані маршрутів
```

---

## Формат файлів перекладу

Кожен файл перекладу експортує об'єкт `messages` типу `Messages`:

```typescript
// src/locales/uk/create.ts
import type { Messages } from '@lingui/core';

export const messages: Messages = {
  // Коментар для групування
  'namespace.section.key': 'Переклад українською',
  'namespace.section.anotherKey': 'Інший переклад',
};
```

### Конвенція іменування ключів

Використовуйте формат: `namespace.section.key`

| Частина | Опис | Приклад |
|---------|------|---------|
| `namespace` | Загальна область (entity або feature) | `request`, `user`, `auth` |
| `section` | Конкретна сторінка або компонент | `create`, `list`, `detail` |
| `key` | Конкретний елемент UI | `title`, `subtitle`, `submitButton` |

> **Важливо:** Завжди використовуйте однакові ключі для обох мов (`uk` та `en`), інакше замість перекладу на екрані з'явиться сам ключ.

## Як додати переклад до нового компонента

### Крок 1: Переконайтеся, що компонент клієнтський
Оскільки `useLingui` це React-контекст, ваш компонент **повинен бути клієнтським**. Додайте директиву `'use client'` на початку файлу.

```typescript
'use client';

import { useLingui } from '@lingui/react';
```

### Крок 2: Створити функцію `t` в компоненті

```typescript
export default function MyComponent() {
  const { i18n } = useLingui();
  const t = (id: string) => i18n._(id);

  return (
    <div>
      <h1>{t('namespace.section.title')}</h1>
      <p>{t('namespace.section.description')}</p>
    </div>
  );
}
```

### Крок 3: Додати ключі перекладу у файли локалей

**Українська (`src/locales/uk/create.ts`):**

```typescript
export const messages: Messages = {
  // ... existing translations
  
  // My new component
  'namespace.section.title': 'Заголовок українською',
  'namespace.section.description': 'Опис українською',
};
```

**Англійська (`src/locales/en/create.ts`):**

```typescript
export const messages: Messages = {
  // ... existing translations
  
  // My new component
  'namespace.section.title': 'Title in English',
  'namespace.section.description': 'Description in English',
};
```

## Як підключити словник до нової сторінки (Route Layout)

Для того щоб `useLingui` працював, дерево компонентів сторінки має бути обгорнуте в `LinguiProvider` з підключеним словником. Зазвичай це робиться на рівні `layout.tsx` відповідного роуту в папці `@src/app/`.

**Приклад `layout.tsx` для нової сторінки:**
```tsx
import type { ReactNode } from 'react';
import { LinguiProvider } from '@/app/providers/LinguiProvider';
import { getLocaleFromHeaders } from '@/locales/locale';

// Імпортуємо словники
import { messages as enMessages } from '@/locales/en/create';
import { messages as ukMessages } from '@/locales/uk/create';

const messagesByLocale = {
  en: enMessages,
  uk: ukMessages,
} as const;

export default async function MyPageLayout({ children }: { children: ReactNode }) {
  // Визначаємо поточну мову користувача на сервері
  const locale = await getLocaleFromHeaders();

  return (
    <LinguiProvider locale={locale} messages={messagesByLocale[locale]}>
      {children}
    </LinguiProvider>
  );
}
```

## Переклад метаданих (Title, Description) сторінок (SEO)

Тексти для `<title>` та `<meta name="description">` **не** використовуються через LinguiProvider, вони живуть окремо у файлах:
- `@src/locales/uk/metadata.ts`
- `@src/locales/en/metadata.ts`

Щоб додати переклад метаданих для нової сторінки:
1. Додайте сторінку в об'єкт `routeMetadata` в обох файлах (`uk` та `en`).
2. На самій сторінці (`page.tsx`) згенеруйте метадані за допомогою серверної функції `generateMetadata`:

**Приклад `page.tsx`:**
```tsx
import { getLocaleFromHeaders } from '@/locales/locale';
import { getMetadataForRoute } from '@/locales/route-metadata';

export async function generateMetadata() {
  const locale = await getLocaleFromHeaders();
  // Передаємо локаль та ключ роуту (наприклад, 'HOME' або 'ABOUT')
  return getMetadataForRoute(locale, 'MY_NEW_ROUTE'); 
}

export default function MyPage() {
  return <div>Вміст сторінки</div>;
}
```

## Спільні переклади (переіспользувані)

Якщо переклад використовується в кількох місцях, використовуйте спільний namespace:

```typescript
// Спільні переклади
'common.loading': 'Завантаження...',
'common.error': 'Сталася помилка',
'common.save': 'Зберегти',
// ...
```

## Динамічні значення в перекладах

Для вставки динамічних значень (наприклад, чисел) можна розділити текст на частини:

```typescript
// Переклади
'request.list.foundPrefix': 'Знайдено',
'request.list.foundSuffix': 'активних запитів',

// Використання в JSX
<>
  {t('request.list.foundPrefix')}{' '}
  <span>{count}</span>{' '}
  {t('request.list.foundSuffix')}
</>
```

## Чеклист для додавання перекладів

- [ ] Компонент клієнтський (`'use client'`)
- [ ] Імпортовано `useLingui` з `@lingui/react` та створено функцію `t` через `i18n._`
- [ ] Замінено всі захардкоджені тексти на `t('key')`
- [ ] Ключі додано в `src/locales/uk/create.ts`
- [ ] Ключі додано в `src/locales/en/create.ts`
- [ ] Формат ключів відповідає конвенції `namespace.section.key`
- [ ] Для нової сторінки створено `layout.tsx` з `LinguiProvider`
- [ ] Для нової сторінки додано переклад SEO метаданих через `generateMetadata` та оновлено файли `metadata.ts`
